How to play Set

The deck has 81 unique cards that vary across four features — each with three possibilities:

  • Number — one, two, or three shapes
  • Shape — diamond, squiggle, or oval
  • Shading — solid, striped, or open
  • Color — red, green, or purple

Every possible combination appears exactly once. When a card shows multiple shapes, all the shapes share the same shape, shading, and color.

What's a "set"?

Three cards form a set if, for each one of the four features, the three cards are either all the same or all different. If two cards match on a feature and the third doesn't, it is not a set.

Game flow

  • The dealer lays out 12 cards. Players race to find a set.
  • There is no turn taking, the game is a race to see who can find a set first.
  • When a set is taken, three new cards are dealt into the empty slots.
  • If nobody can find a set, every player can vote for a Hint — once everyone votes the server either reveals a card from a real set or, if there isn't one, deals 3 more cards.
  • The board can grow from 12 to 15 to 18 to 21 cards. Mathematically a set must exist among any 21 cards, so it never grows further.
  • The game ends when the deck is empty and no sets remain on the table. The player with the most sets wins.

Multiplayer UX cheat sheet

Lobby & identity

  • The Lobby lists every open game with name, player count, and elapsed time.
  • It also lists the leaderboard for the top players showing top points score and # wins. The score is weighted by how many people you beat in each game, so beating 10 players is worth more than beating 2 players, even though both are just "1 win". You only get a score of zero for playing against yourself.
  • Each game has its own page — the URL is shareable / bookmarkable.
  • Leave Game sends you back to the lobby. Your row stays in the players panel, dimmed and tagged LEFT; if you rejoin, your score continues.

Selecting cards

  • Click a card to select it. After three are selected the server checks for a set automatically.
  • Selections are private. What you've clicked is invisible to other players, and other players' actions never modify your selection — with one exception below.
  • Click the page background to clear your current selection.
  • An invalid set shows a red message explaining the first failing property; the cards stay highlighted until you click somewhere else.

Hint (distributed vote)

  • Hint is a vote, not an instant reveal. The badge (e.g. 1/3) shows how many players have voted in the current round.
  • When everyone votes, the next card from a real set is added to everyone's selection.
  • You can build on this selection hint or select other cards if you want.
  • If every one asked for three hints then those cards are removed and nobody scores for that round.
  • If the board has no valid set, the agreed upon hint will deal 3 more cards causing the hand to grow.

Race fairness

  • The server measures each player's ping time and waits a short window so the player who actually clicked first wins, regardless of who's on the slowest connection.
  • If someone beats you to a set you submitted, you'll see "<winner> beat you, sorry".
  • Two non-overlapping sets submitted in the same window can both succeed.

Scoreboard & broadcasts

  • The Players strip is sorted by sets found. Your row is highlighted; the most-recent ping (in ms) appears as a small pill next to each player.
  • A scoring set pulses the three matched cards in yellow on every screen, then the cards flip away and the scorer's row sparkles.
  • Players who go silent for more than 30 seconds are dimmed automatically and they no long need to vote for hints; same is true for players who leave the game.

Game over & new round

  • When the deck empties and no sets remain, an overlay shows the winner, your final count, and elapsed time.
  • New Round wipes the board, reshuffles, and resets every player — everyone auto-rejoins so the next game starts truly fresh.
  • Export Game downloads a JSON file with the initial deck shuffle and every accepted move, so the round can be replayed or analysed offline.

About the game

SET is a real-time card game designed by Marsha Falco in 1974 and published by Set Enterprises in 1991. There is some pretty interesting mathematics about the game available on https://en.wikipedia.org/wiki/Set_(card_game) including a proof by Donald Knuth found in 2001 that the maximum number of cards that can be laid out without a valid set is 20.

Application Architecture

This game is show-case example of the MSR Garnet in-memory cache and in particular how it can be used as an optimized store for game state and real-time pub/sub eventing across all web clients that have joined the same game, for super fast real time game play, which is required for this type of game that is a race (i.e. there is no turn taking).

See Software Architecture. Source code is available at https://github.com/msr-central/setgame/. The game is implemented entirely in C# using .NET 10.0 with a single ASP.NET Core server hosting the game web UI, and the back end game state apis that handle all game logic and rules enforcement. The served web client is a three page application implemented in vanilla JavaScript, one for game lobby, one for the game itself, and this one for help. The game client uses the JavaScript SSE EventSource API to subscribe to real-time game events broadcast from the server hosted at /api/games/${id}/events.
